## Decisions
- Phase 2 task 2.3 (2026-07-09): **Razorpay** in `modules/payments`. Webhook raw-body ordering: `app.use('/api/v1/webhooks', express.raw({type:'*/*'}), webhookRoutes)` mounts **before** `express.json()` (and before the global limiter — gateways retry). `POST /payments/razorpay/order` (requireAuth) guards the held order (owner/PENDING/not-expired/not-free/INR-only), sets `order.gateway=RAZORPAY`, creates a Razorpay order + a Payment(CREATED) row, returns `{razorpayOrderId, amount(paise), keyId}`; 503 when unconfigured. `POST /payments/razorpay/verify` checks `HMAC_SHA256(order_id|payment_id, KEY_SECRET)` (timing-safe) and **records the attempt but never fulfils** (webhook does). `POST /webhooks/razorpay` verifies `HMAC_SHA256(rawBody, WEBHOOK_SECRET)` (400 on mismatch), links payment→order via `Payment.gatewayOrderId`, then **payment.captured** → idempotency (already-PAID → 200), **amount-mismatch guard** (ignore, never fulfil under/overpayment), Payment CAPTURED + `markPaidAndFulfil` (the conditional flip gates single fulfilment); **payment.failed** → Payment FAILED + `releaseHeldOrder(FAILED)`. Verified headlessly with dummy secrets + locally-signed payloads; **live order creation + real dashboard webhooks need the user's test keys (EXIT sign-off)**. Client checkout wiring is 2.6. `razorpay` dep added.
- Phase 2 task 2.2 (2026-07-09): **`POST /orders` (§8.1)** validates event PUBLISHED+future + each ticket type active/in-window/qty within min–max, **recomputes all money server-side** (client sends only `{ticketTypeId, quantity}` + optional `promoCode` — no prices to spoof), reserves inventory in **one `withTransaction`** with the `$expr` `$lte[$add[$quantitySold,qty],$quantityTotal]` guard (`modifiedCount!==1` → abort → **409 SOLD_OUT**; multi-item aborts roll back earlier increments), then inserts the **PENDING** order (embedded item snapshots + `expiresAt = now + ORDER_HOLD_MINUTES`). `orderNumber` from `nextSeq('order')` is generated **before** the txn (a retry/abort only skips a number). **Gateway at creation** = `FREE` if total 0, else `RAZORPAY` (INR) / `STRIPE` (non-INR) as a default; the chosen gateway is set when payment is initiated (2.3/2.4). **Free order (total 0, incl. a 100%-off promo)** → `markPaidAndFulfil` flips PENDING→PAID + creates tickets + REGISTRATION_CONFIRMATION, no gateway. **`releaseHeldOrder(orderId, toStatus)`** re-checks PENDING inside a txn then flips + decrements `quantitySold` — shared by cancel (`POST /orders/:id/cancel`, owner + PENDING-only), the expiry cron, and (2.3/2.4) the failed-payment webhook. **Expiry cron** `expireOrders` (node-cron `*/5 * * * *`, started in index.js only) → PENDING past `expiresAt` → EXPIRED + restore; idempotent (conditional flip). `GET /me/orders`(+`/:id`) list/detail with ownership. Fee = `round((subtotal−discount)×SERVICE_FEE_PERCENT/100)`; SERVICE_FEE_PERCENT=5, ORDER_HOLD_MINUTES=15 from env. node-cron added.
- Phase 2 sequencing (2026-07-09): **fulfilment is ONE shared `fulfillOrder()`** used by the free-order path (2.2) and both webhooks (2.3/2.4).
